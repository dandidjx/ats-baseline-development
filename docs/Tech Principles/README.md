# Tech Principles

This folder documents the technical principles and implementation standards for the platform.

## Files
- `01-architecture-overview.md`: target architecture and system boundaries.
- `02-data-types.md`: data type strategy and schema expectations.
- `03-modules-forms.md`: module/form design guidance.
- `04-modules-workflows.md`: workflow module principles.
- `05-standards.md`: coding, API, localization, and configuration standards.

## Notes
- Use these files to align implementation decisions with architecture constraints.
- `_archived/` content is intentionally ignored and is not part of active standards.
